======
"Rc"
======

---------------------------------
Multiple Ownership with "Rc<T>"
---------------------------------

- Useful when single value is owned by multiple parts of a program
  
  - Only provides *immutable access* to the data

  - Tracks number of active references

  - Prevents data cleanup until last owner finishes
  
- Called :dfn:`Reference Counted (Smart) Pointer`

  - Not in prelude
  
.. code:: rust   

  use std::rc::Rc;

---------------------------
"Rc<T>" counts references
---------------------------

- Shares ownership of value on the heap using :rust:`Clone`
    
  - Creates a *shallow* copy not a deep copy

    - Only the *pointer* is copied
	
  - Increments the internal counter 
 
.. code:: rust 

  // Both 'var_a' and 'var_b' share ownership of the value
  let var_a = Rc::new(5);
  println!("Count: {}", Rc::strong_count(&var_a)); 

  let var_b = Rc::clone(&var_a);
  println!("Count: {}", Rc::strong_count(&var_a)); 
  
:command:`Count: 1`

:command:`Count: 2`

------------------
Shared Ownership
------------------

- :rust:`Box<T>` allows only one owner at a time

.. code:: rust 

  let tic = Box::new(5);
  let tac = tic;
  let toe = tic;

:error:`error[E0382]: use of moved value: 'var_a'`
  
- :rust:`Rc<T>` allows multiple ownership but no mutable access

.. code:: rust 

  let tic = Rc::new(5);
  let tac = Rc::clone(&tic);
  let toe = Rc::clone(&tic);  
  
  *var_a += 10; //Error: no mutable access
  
:error:`error[E0594]: cannot assign to data in an 'Rc'`

--------------
Fair Warning
--------------

- Works for :rust:`Copy` types

.. code:: rust

  let tic = 5;
  let tac = Box::new(tic);
  let toe = Box::new(tic);
  
- :rust:`Clone` works also with :rust:`Box<T>`

.. code:: rust

  let tic = Box::new(5);
  let tac = Box::clone(&tic);
  let toe = Box::clone(&tic);

.. warning::  

  In **both** cases, data is **not shared**, it is **duplicated**

------------------------------------------
Why Use "Rc<T>" if "Box<T>" Can "Clone"?
------------------------------------------

**Avoids expensive data duplication**

.. list-table::
   :header-rows: 1
   
   * - **Smart Pointer**
     - **Ownership**
	 - **Access**
	 - **Main Use Case**

   * - :rust:`Box<T>`
     - Single
	 - Mutable
     - Pointer for big datas or recursive types

   * - :rust:`Rc<T>`
     - Multiple
	 - Immutable
     - Sharing data across many parts of a program
